HTML5




Let's start by HTML5 and its history,
The first version of HTML was written by Tim Berners-Lee in 1993. Since then, there have been many versions of HTML. The most widely used version throughout the 2000's was HTML 4.01, which became an official standard in December 1999. After alot of versions it was either XHTML2 or HTML5, and here the choice of HTML5 was dictatorious by Lan Hickson.



What is different in HTML5?
Without going deep in details,

  • Shortened DOCTYPE.
  • New semantic tags.
  • Native audio and video.
  • Creating complex games/graphics with canvas.
  • Access to Javascript APIs.
  • No need to meaningless class names.

HTML5 Multimedia




Multimedia comes in many different formats. It can be almost anything you can hear or see, examples: Pictures, music, sound, video, animation...
Web pages often contain multimedia elements of different types and formats.

Videos
Before HTML5, there was no standard for showing videos on a web page, videos could only be played with a plug-in (like flash).
The HTML5 <video> element specifies a standard way to embed a video in a web page

Audios
Text between the <audio> and </audio> tags will display in browsers that do not support the audio element.
Multiple <source> elements can link to different audio files.







CSS3







Why start using CSS3 along with HTML5?

  • More robust pages.
  • More playing for designers and developers.
  • More exciting landscape to attract new clientele.
  • Reduced development time.
  • No need the many hacks and workarounds used before.





Characteristics of modern CSS websites:

























Progressively enhanced

It is a strategy that puts emphasis on web content first, it involves separating the presentation semantics from the content, with presentation being implemented in one or more optional layers, activated based on aspects of the browser or Internet connection of the user.


Adaptive to diverse users

Web pages have to accommodate the diverse range of browsers, devices, screen resolutions, font sizes, assistive technologies and other factors that users bring to the table. so it is important to consider the browser, the platform, the devices, the screen resolution and other factors.


Modular

Modular CSS can be broken down into chunks that work independently to create design components that can themselves. be reused independently.


Efficient

A website is effective when both the users and the owners achieve goals for the site. For the users, normally some kind of sequence is involved. First, users must know about the site, then they find the site on the web, then they find the page they want, with the information they want.


Typographically rich

concept of creating readable yet uniquelooking text by applying tried and true typographic principles using the newest technologies. Typography is one of the most rapidly evolving areaS of Web design.
